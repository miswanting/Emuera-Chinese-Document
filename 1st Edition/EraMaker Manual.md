## CSV文件格式

> 来自：http://cbaku.com/b/erakanon/eramacsv.html

仅仅看本文很难获得全貌，推荐一边改一边试。

与`EraMaker.exe`的同一目录下，有一个名为`CSV`的文件夹，其中的CSV文件常见为：

| 文件名称     | 描述             |
| ------------ | ---------------- |
| GameBase.csv | 注册游戏基本数据 |
| Palam.csv    | 注册训练参数     |
| Abl.csv      | 注册能力         |
| Talent.csv   | 注册天赋         |
| Mark.csv     | 注册刻印         |
| Exp.csv      | 注册经验         |
| Train.csv    | 注册命令         |
| Item.csv     | 注册物品         |
| Str.csv      | 注册文本         |
| CharaXX.csv  | 注册角色初始数据 |

### 文件格式

#### 通用格式

- 第一行的第一个字符是半角; 如果用分号表示，则忽略该行。 空行将被忽略。

- ```
  ;体力与气力设置
  基础,0,2000
  基础,1,1000
  
  ;能力设置
  能力,0,2
  ```

- 类型为数字时，请使用半角。

- ```
  ;正确
  121,扶她
  
  ;错误
  １２１,扶她
  ```

- 字符串不用双引号`""`包括起来，而是直接写入。

- ```
  ;正确
  0,顺从
  1,欲望
  2,技巧
  
  ;错误
  0,"顺从"
  1,"欲望"
  2,"技巧"
  ```

详见CSV文件格式手册

#### GameBase.csv 专用格式

第一列为指令，第二列及之后为数据

| 名称       | 格式                        | 描述                                                         |
| ---------- | --------------------------- | ------------------------------------------------------------ |
| 游戏识别码 | コード,`数字`               | 代表游戏的数字码。用于防止意外加载另一个游戏的存档。         |
| 版本号     | バージョン,`字符串`         | 将游戏版本设置为数字。 屏幕上显示（数字）除以 1000（100 为 0.10）。 最初，不会加载以不同版本保存的保存的游戏。 |
| 标题       | タイトル,`字符串`           | 将游戏标题设置为字符串。 在启动时显示。                      |
| 作者名     | 作者,`字符串`               | 将游戏的作者设置为字符串。 在启动时显示。                    |
| 发布时间   | 製作年,`字符串`             | 将游戏的制作年份设置为字符串。 在启动时显示。 2005-2006 等表示法也是可能的，因为它不是数字。 |
| 附加信息   | 追加情報,`字符串`           | 将游戏的其他信息设置为（字符串）。 在启动时显示。            |
| 初始对象   | 最初からいるキャラ,`数字`   | 游戏开始时，设置角色，而不是英雄。 与 EraLight 一样，它用于游戏，从一开始就确定要训练的角色。 （数字）是字符编号。 例如，如果指定 Chara01.csv，则指定 12，然后转到 Chara12.csv。 |
| 无项目     | アイテムなし,`数字`         | 如果为1，则允许在没有项目的情况下训练，即使在训练时需要物品也可以。当你创建一个游戏时，没有物品时，请设置为1。 |
| 兼容版本号 | バージョン違い認める,`数字` | 如果存档的游戏版本大于或等于`兼容版本号`，则即使版本不同，也可以读取该存档。 |

示例

```
;版本1.20后保存的游戏读取
バージョン違い認める,1200
```

#### Palam.csv 专用格式

第一列为参数编号，第二列为参数名称。

建议编号从 0 开始，中间不要中断。

编号范围为0~99。

#### Abl.csv 专用格式

第一列为能力编号，第二列为能力名称。

建议编号从 0 开始，中间不要中断。

编号范围为0~99。

#### Talent.csv 专用格式

第一列为天赋编号，第二列为天赋名称。

编号可以被中断。

编号范围为0~99。

#### Mark.csv 专用格式

第一列为刻印编号，第二列为刻印名称。

建议编号从 0 开始，中间不要中断。

编号范围为0~99。

#### Exp.csv 专用格式

第一列为经验编号，第二列为经验名称。

编号可以被中断。

编号范围为0~99。

#### Train.csv 专用格式

第一列为训练编号，第二列为训练名称。

编号可以被中断。

编号范围为0~99。

#### Item.csv 专用格式

第一列为物品编号，第二列为物品名称，第三列为物品价格。

编号可以被中断。

编号范围为0~99。

#### Str.csv 专用格式

第一列为文本编号，第二列为文本。

文本的长度没有限制。

编号可以被中断。

编号范围为0~19999。

#### 关于 Str.csv 中的`***`和`+++`

示例游戏 EraLight 的 Str.csv 中会显示许多`***`和`+++`。在游戏中，`***`会被转换为“あかり”和“浩之ちゃん”。
理解这个特性需要一些基础的 EraBasic 知识，但你可以先简单的记下一些简写符号：

- ***：被训练的人的`姓名`
- +++：玩家的`称呼`
- $$$：被训练的人的`称呼`
- ///：助手的`姓名`
- ===：助手的`称呼`

## EraBasic语法

> http://cbaku.com/b/erakanon/eramaerc.html

### 启动EraBasic

在 EraMaker 中，当您启动游戏时，将显示标题屏幕，并可以选择“从头开始”和“加载存档”。 如果在此处选择"从头开始"，则会触发 ERB 文件中的 EVENTFIRST 函数。
当 EVENTFIRST 结束时，游戏将会停止。因此，在 BEGIN 中，您需要调用 SHOP 或 TRAIN 等。
在示例游戏中，请参阅 SYSTEM.ERB 的开头。

### SHOP

#### 进入商店

进入 SHOP 时，将调用@EVENTSHOP函数（如果有）。 这是一个事件函数。然后调用名为@SHOW_SHOP的函数。 在此处查看基本信息，如金额和训练字符等、按PRINT_SHOPITEM显示销售、保存和加载等特殊操作。

#### 在SHOP中选择命令

如果选择了 0 到 99，则意味着购买项目。 如果选择了其他数字，则调用名为@USERSHOP的函数。 所选数字存储在 RESULT 中，因此请执行适当的操作。
\- 如果看到示例游戏 SHOP.ERB，它们都会更容易理解。

#### 购买物品

当你买东西时，@EVENTBUY函数（如果有）被调用。 这是[一个事件函数](http://cbaku.com/b/erakanon/eramaerc.html#eventfunc)。
（您购买的物品将从商店的分类中消失，因此您可能需要在此处执行这些操作）

### TRAIN

##### 进入训练

进入 TRAIN 时，将调用一个函数@EVENTTRAIN（如果有）。 这是[一个事件函数](http://cbaku.com/b/erakanon/eramaerc.html#eventfunc)。
然后调用@SHOW_STATUS函数。 在这里，请执行基本信息，如日期和检查字符，以及显示PRINT_PALAM训练参数。
然后，系统会自动显示您可以运行的命令。 调用函数@COM_ABLExx所有命令，如果 RETURN 1 是可执行的。 这意味着，如果相应的@COM_ABLExx不存在，则执行。 查看示例游戏的 COMABLE.ERB 会很有帮助。
此外，调用@SHOW_USERCOM函数。 在此处查看特殊命令，如结束训练。
\- 如果查看示例游戏 SYSTEM.ERB 和 INFO.ERB，则两者都很容易理解。

##### 在 TRAIN 中选择命令

当玩家选择命令时，首先调用@EVENTCOM函数（如果有）。 这是[一个事件函数](http://cbaku.com/b/erakanon/eramaerc.html#eventfunc)。
此外，将调用与所选命令对应的函数。 例如，选择"正常位置"和 TRAIN。 如果在 CSV 中指定了"正常位置"命令编号 20，则调用@COM20函数。
•对于像这样的命令，根据角色的能力可能无法执行。 在这种情况下，在@COMxx中间调用 RETURN 0。 然后，它返回到命令选择，而不运行该命令。
如果可以运行该命令，则从@COMxx调用 RETURN 1。 然后，@SOURCE_CHECK调用函数。 训练结果现在反映在训练参数中。
• 如果查看示例游戏的 COMxx.ERB 和 SOURCE.ERB，则都很容易理解。

##### 在 TRAIN 中退出命令选择

最后，调用@EVENTCOMEND的函数（如果有）。 这是[一个事件函数](http://cbaku.com/b/erakanon/eramaerc.html#eventfunc)。
执行命令的字符的台词等可能在这里执行。 我认为，如果你看看样品游戏era light的CKOJOxx.ERB，这是很容易理解的。

##### 用户命令

如果所选命令没有相应的@COMxx，则调用@USERCOM。 由于 RESULT 包含所选数量，因此请执行相应的操作。
• 查看示例游戏的 SYSTEM.ERB 会更容易理解。

### AFTERTRAIN

进入 AFTERTRAIN 时，将调用@EVENTEND函数（如果有）。 这是[一个事件函数](http://cbaku.com/b/erakanon/eramaerc.html#eventfunc)。
• 训练结束后角色的台词（扔掉的台词？ 最好在这里处理它。 此外，应该在这里做珠子的计算，通过训练获得。 查看示例游戏的 AFTERTRA.ERB 会很有帮助。

### ABLUP

#### 显示ABLUP

首先，@SHOW_JUEL函数。 看看你有一颗珠子。
然后，@SHOW_ABLUP_SELECT函数。 查看容量列表、退出命令等。
• 查看示例游戏的 ABL.ERB 会更容易理解。

#### 选择 ABLUP 命令

当玩家选择命令时，将调用与所选命令对应的函数。 例如，如果选择了 [3] 的"C 感觉"，则调用@ABLUP3函数。
如果所选命令没有相应的@ABLUPxx，则调用@USERABLUP。 由于 RESULT 包含所选数量，因此请执行相应的操作。
• 查看示例游戏的 ABLUPxx.ERB 和 ABL.ERB 会很有帮助。

### TURNEND

称为@EVENTTURNEND的函数（如果有）。 这是[一个事件函数](http://cbaku.com/b/erakanon/eramaerc.html#eventfunc)。
• 最好处理时间进度和体力恢复。
*请注意，如果没有@EVENTTURNEND，它就会停止。
• 查看示例游戏的 SYSTEM.ERB 会更容易理解。

### 事件函数

#### 什么是事件函数

事件函数是游戏中的某个时间调用的函数。 例如，当您完成命令或开始训练时，
事件函数可用于显示角色的行。 编写直接在函数（如@COMxx）上显示行的指令后，可能难以理解，但可以使用事件函数进行隔离和管理。
事件函数的性质可以有多个具有相同名称的函数。

示例

```
;正常位を実行し終わったときのセリフ
@EVENTCOMEND
;20が正常位とする。20でなければ無視
SIF SELECTCOM != 20
   RETURN 0
;FLAG:1000が0でなければ無視
SIF FLAG:1000
   RETURN 0
PRINTW 「や・・・生はいやあっ！」
;FLAGを設定する。一度表示されるとその後表示されない
FLAG:1000 = 1
RETURN 1

;背後位を実行し終わったときのセリフ
@EVENTCOMEND
;21が背後位とする。21でなければ無視
SIF SELECTCOM != 21
   RETURN 0
;FLAG:1001が0でなければ無視
SIF FLAG:1001
   RETURN 0
PRINTW 「こんな格好で・・・」
;FLAGを設定する。一度表示されるとその後表示されない
FLAG:1001 = 1
RETURN 1
```

事件函数可以具有"属性"。
•#SINGLEの性質を付与すると，如果以 RETURN 1 结尾，则即使函数同名，也不会执行更多操作。 当角色一次说出多个行时，它很有用，例如，当不自然时。
\#PRIの性質を付与すると，它先运行于其他同名函数之前。 死亡检查等，是有效的处理，变得不自然，如果你不先做。
\#LATERの性質を付与すると，它在其他同名函数之后执行。 它是有效的处理，变得不自然，如果你不做最后，如显示"一天结束了"。

示例

```
@EVENTTURNEND
#SINGLE
SIF FLAG:1000
   RETURN 0
FLAG:1000 = 1
PRINTW 「もういやだ・・・」
RETURN 1

@EVENTTURNEND
#SINGLE
SIF FLAG:1001
   RETURN 0
FLAG:1001 = 1
PRINTW 「家に帰して・・・」
RETURN 1

@EVENTTURNEND
#SINGLE
SIF FLAG:1002
   RETURN 0
FLAG:1002 = 1
PRINTW 「外に出たい・・・」
RETURN 1

@EVENTTURNEND
#LATER
PRINTW 一日が終わった・・・
RETURN 1

```

（上の３つのセリフは一度に１つずつしか表示されない。「一日が終わった・・・」は#LATERがついているので、FLAG:1000とFLAG:1001とFLAG:1002が1になるまでは一度も表示されない）

### 关于 ERB 文件

在 eramaker.exe 的正下方，将名为 ERB 的文件夹放在其中 ERB 文件中。
扩展是. 可以是任何文件名，如果 ERB 是。
请编辑文本编辑器，如记事本。

### 如何编写 ERB 文件

#### 注释与空格

在所有 ERB 文件中

，第一个字符是半角; 如果用分号表示，则忽略该行。 空行将被忽略。
不要在行尾用分号或注释。
\- 即使行首的空格和制表符被忽略，

```
正
;お金の設定
MONEY = 500
   時刻の設定
   DAY = 10
   TIME = 1
;ゲーム開始
PRINT どうする？

誤
MONEY = 500;
TIME = 5;（5日目からスタート）
```

#### 输入半角

输入数字、指令、变量名称、函数名称等时，请使用两个角。

```
正
MONEY = 500
PRINT ゲームをはじめます。
誤
ＭＯＮＥＹ ＝　５００
ＰＲＩＮＴ ゲームをはじめます。
```

#### 输入字符串

当字符串""来时，它无法正常工作。

```
正
PRINT 一日が終わった・・・
誤
PRINT "一日が終わった・・・"
```

#### 一行完成

写长指令时，不要超过两行。

```
正
PRINT 名雪の身体は敏感だったし経験も豊富だったから、多少乱暴にしても悦ぶ。口では否定していても、名雪はどんどん乱れていく。だが、眠っているという特殊な状況が俺に大胆な動きをためらわせた。

誤
PRINT
名雪の身体は敏感だったし経験も豊富だったから、多少乱暴にしても悦ぶ。口では否定していても、名雪はどんどん乱れていく。だが、眠っているという特殊な状況が俺に大胆な動きをためらわせた。
```

## EraBasic变量列表

### 基本变量

#### 通用变量

##### A-Z

从 A 到 Z 是字母表中的单个字符的变量。
\* 请使用尽可能多的一次性数据。 但是，它不适合长时间数据存储，因为我们不知道在哪里重写它。

##### 计数

使用 REPEAT 指令时，计算重复次数的变量。
• 避免在 REPEAT_REND 内部重写 COUNT 的内容，因为这将导致故障。



##### RESULT

记录各种结果的变量。
由于我们不知道在哪里重写，因此最好将数据传输到另一个变量进行处理，但在现场使用时除外。



##### RESULTS（字符串变量）

记录各种结果的字符串变量。
由于我们不知道在哪里重写，因此最好将数据传输到另一个变量进行处理，但在现场使用时除外。

#### 基本信息变量

##### 日

记录日期。 单位等尚未确定，所以你可以自由处理它。

##### 时间

记录时间。 单位等尚未确定，所以你可以自由处理它。

##### 钱

记录资金。 程序在购物时会参考，因此，除非您得到或丢失了资金，否则不要重写它。

#### 训练基础信息变量

##### 大师

\- 指英雄[角色的注册号](http://cbaku.com/b/erakanon/eramavar.html#enroll)。 请注意，它们可能与 CharaXX.csv 中指定的数字不同。 通常为 0。

##### 目标

指正在训练[的字符的注册号](http://cbaku.com/b/erakanon/eramavar.html#enroll)。 请注意，它们可能与 CharaXX.csv 中指定的数字不同。

##### ASSI

指助理[角色的注册号](http://cbaku.com/b/erakanon/eramavar.html#enroll)。 请注意，它们可能与 CharaXX.csv 中指定的数字不同。

##### PLAYER

指正在训练的人类[角色的注册号](http://cbaku.com/b/erakanon/eramavar.html#enroll)。 通常，它应该与 MASTER 或 ASSI 匹配。 请注意，它们可能与 CharaXX.csv 中指定的数字不同。

##### 查纳姆

指当前注册的字符数。 包括英雄。 用户无法更改此变量。

## 字符的注册号

#### 字符的注册号和字符号之间的区别是什么？

##### 无实体字符数据

• CharaXX.CSV 指定了"数字"。 此处指定的数字是字符编号。
但是，CharaXX.CSV 中的角色在训练开始时没有实体。 当指令 ADDCHARA 被调用时，它第一次具有实体。

##### 注册角色

\- 在游戏开始时，只有英雄有一个实体。 主人公的"字符注册号"为0。
\* 假设字符是"字符编号"是 5 在这里在 ADDCHARA 注册。 这个角色的"字符号"是5，但"字符的注册号"是1，因为它成为英雄的下一个。 此外，如果另一个"字符号"包含 7 个字符，则该字符的注册号为 2。

##### 删除字符

现在，假设DELCHARA命令"字符的注册号"删除了1个字符。 然后，我稍后添加的"字符编号"7 字符的"注册号"为 1。
因此，请记住，"字符的注册号"将从 0 中授予当前注册字符。



## 关于污垢

#### 污垢数据的特殊性

##### 污垢类型

污垢数据由 STAIN 管理。 STAIN：TARGET：0 是指训练角色口中的污垢。
然而，这里考虑的污垢类型有很多种。 如果，精子的污垢在雷兹训练中，如果昆尼被做，的污垢就会被制造出来。
在示例游戏中，我们假设四种类型的污垢：瓦吉纳（）、迪克、暨和肛门。

##### 如何描述脏数据

现在，对于这个污垢，数字1，2，4，8被分配。 换句话说，如果肛门和暨有污垢在4+812。 如果和暨有污垢，它是条件，如 5 在1⁄4。
如果你这样表示，你可以用一个变量处理四种类型的污垢。 然而，暨污渍添加到嘴里和确定口腔中是否有污渍等在传统的 + - */ %中不起作用。

##### 如何确定脏数据

因此，我们将使用 "&"符号。 &amp;（和）和 | （或）类似，但单独使用。
例如，假设 STAIN：TARGET：0 为 12。 在这里STAIN：目标：0 &4 表示仅提取 4 个部分。 即 STAIN：TARGET：0 & 4 = 4。
接下来，假设 STAIN：TARGET：0 是 1+2+8=11。 在这里STAIN：目标：0 &4 表示仅提取 4 个部分。 即 STAIN：TARGET：0 & 4 = 0。
这样，您可以轻松地知道是否有污垢。

##### 如何添加脏数据

接下来，STAIN：TARGET：假设 0 是 1⁄4=5。 STAIN： TARGET：0 | 2，添加 2 部分。 即 STAIN：TARGET：0 | 2 = 7。
STAIN：假设目标：0 是 1+2+4=7。 STAIN： TARGET：0 | 2，添加 2 部分。 但是，这两个部分已经包含，因此不会更改。 即 STAIN：TARGET：0 | 2 = 7。
STAIN： TARGET：0 | 也可以写成2。
这样，您就可以对污垢进行额外的处理，而不必担心污渍是否已经包含。
